## 2026-04-14 - Implementing Form Loading States
**Learning:** Preventing double submissions in form updates (like `Settings.tsx`) significantly improves user experience. In this design system, setting a boolean state (`isSaving`) tied directly to `disabled={isSaving}` on both form inputs (`input`, `textarea`) and the submit button, alongside disabled tailwind classes (`disabled:opacity-50 disabled:cursor-not-allowed`) and a `Loader2` spinner ensures robust protection against accidental multiple clicks and gives users clear visual feedback while operations are pending. This acts as a standard pattern to handle generic async operations.
**Action:** Always add visual feedback alongside disabling entire form blocks during database interactions to prevent unwanted behavior and reassure the user. Use `lucide-react`'s `Loader2` and tailwind's `disabled` pseudo-class for consistent styling.
